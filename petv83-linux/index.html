Nmap: Network Mapper Mastery

What is Nmap?

Nmap (Network Mapper) is a free and open-source network discovery and security auditing tool. Created by Gordon "Fyodor" Lyon in 1997, it has become the de facto standard for network reconnaissance and is used by security professionals, system administrators, and network engineers worldwide.

Core Capabilities

  • • Host discovery and network mapping
  • • Port scanning and service detection
  • • Operating system fingerprinting
  • • Network vulnerability assessment
  • • Firewall and IDS evasion techniques
  • • Scriptable network discovery (NSE)
  • • Performance and timing optimization
  • • Comprehensive reporting and output

Nmap Fundamentals

Basic Syntax

nmap [Scan Type] [Options] [Target Specification]

Your First Nmap Scan

1
Basic Host Discovery
nmap 192.168.1.1

This performs a basic scan of the target IP address, checking the most common 1,000 ports.

Nmap Scripting Engine (NSE)

The Nmap Scripting Engine (NSE) is one of Nmap's most powerful features, allowing you to write and execute custom scripts for network discovery, vulnerability detection, and exploitation.

NSE Script Categories

  • auth: Authentication bypass
  • broadcast: Network broadcast discovery
  • brute: Brute force attacks
  • default: Default scripts
  • discovery: Service/host discovery
  • dos: Denial of service detection
  • exploit: Exploit known vulnerabilities
  • external: External service queries
  • fuzzer: Fuzzing attacks
  • vuln: Vulnerability detection

Creating Custom NSE Scripts

Here's a simple example of a custom NSE script:

-- Custom banner grabbing script
local shortport = require "shortport"
local stdnse = require "stdnse"

description = [[
Attempts to grab banners from common services
]]

author = "Your Name"
license = "Same as Nmap--See https://nmap.org/book/man-legal.html"
categories = {"discovery", "safe"}

portrule = shortport.port_or_service({21, 22, 23, 25, 53, 80, 110, 143, 993, 995}, {"ftp", "ssh", "telnet", "smtp", "domain", "http", "pop3", "imap", "imaps", "pop3s"})

action = function(host, port)
local socket = nmap.new_socket()
socket:set_timeout(5000)

local status, err = socket:connect(host, port)
if not status then
return stdnse.format_output(false, "Failed to connect: " .. err)
end

local status, banner = socket:receive_lines(1)
socket:close()

if status then
return stdnse.format_output(true, "Banner: " .. banner)
else
return stdnse.format_output(false, "No banner received")
end
end

Firewall and IDS Evasion

Important: These techniques should only be used during authorized penetration testing or on your own networks. Using evasion techniques on unauthorized networks may be illegal.

Packet Fragmentation

# Fragment packets to evade simple packet filters
nmap -f 192.168.1.100

# Use smaller fragments
nmap -ff 192.168.1.100

# Custom MTU size
nmap --mtu 24 192.168.1.100

Fragmentation can help bypass simple packet filters that don't reassemble packets.

Decoy Scanning

# Use decoy IP addresses
nmap -D 192.168.1.5,192.168.1.6,192.168.1.7 192.168.1.100

# Random decoys
nmap -D RND:10 192.168.1.100

# Specify your real IP in decoy list
nmap -D 192.168.1.5,ME,192.168.1.7 192.168.1.100

Decoy scanning makes it difficult to determine which IP is conducting the actual scan.

Source Port Manipulation

# Use specific source port (common: 53, 80, 443)
nmap --source-port 53 192.168.1.100

# Short form
nmap -g 53 192.168.1.100

# Common bypasses
nmap -g 80 192.168.1.100 # HTTP source port
nmap -g 443 192.168.1.100 # HTTPS source port
nmap -g 53 192.168.1.100 # DNS source port

Timing Evasion

# Very slow scanning to avoid detection
nmap -T0 192.168.1.100

# Custom timing controls
nmap --scan-delay 5s 192.168.1.100
nmap --max-scan-delay 10s 192.168.1.100

# Randomize host scan order
nmap --randomize-hosts 192.168.1.0/24

Advanced Evasion Combinations

# Comprehensive stealth scan
nmap -sS -T1 -f --data-length 200 -D RND:10 --source-port 53 192.168.1.100

# Explanation of above command:
# -sS: SYN scan (stealth)
# -T1: Sneaky timing
# -f: Fragment packets
# --data-length 200: Add random data to packets
# -D RND:10: Use 10 random decoy IPs
# --source-port 53: Use DNS source port

Real-World Test Cases

Test Case 1: Corporate Network Assessment

Scenario: You're conducting an authorized penetration test on a corporate network (10.0.0.0/16).

Step-by-Step Process:
1
Initial Network Discovery
# Discover live hosts
nmap -sn 10.0.0.0/16 -oA network_discovery

# Extract live IPs
grep "Nmap scan report" network_discovery.nmap | awk '{print $5}' > live_hosts.txt
2
Port Scanning
# Scan top 1000 ports on all live hosts
nmap -sS -T4 --top-ports 1000 -iL live_hosts.txt -oA port_scan

# Focus on critical servers
nmap -p- 10.0.0.100 -oA full_scan_server
3
Service Enumeration
# Detailed service detection
nmap -sV -sC --open -iL live_hosts.txt -oA service_detection

# Focus on web services
nmap -p 80,443,8080,8443 --script http-enum -iL live_hosts.txt
4
Vulnerability Assessment
# Run vulnerability scripts
nmap --script vuln --open -iL live_hosts.txt -oA vulnerability_scan

# Specific vulnerability checks
nmap --script ms17-010,eternal-blue -p 445 -iL live_hosts.txt

Test Case 2: Web Application Security Testing

Scenario: Testing a web application for security vulnerabilities.

# Comprehensive web app scan
nmap -p 80,443,8080,8443 --script http-enum,http-headers,http-methods,http-robots.txt webapp.example.com

# SSL/TLS security
nmap -p 443 --script ssl-enum-ciphers,ssl-cert,ssl-date webapp.example.com

# Common vulnerabilities
nmap -p 80,443 --script http-vuln* webapp.example.com

Wireshark: Network Analysis Mastery

What is Wireshark?

Wireshark is the world's foremost and widely-used network protocol analyzer. Originally developed by Gerald Combs in 1998 as "Ethereal," it provides deep inspection of hundreds of protocols and runs on multiple platforms including Windows, macOS, and Linux.

Core Capabilities

  • • Live packet capture and offline analysis
  • • Deep packet inspection and protocol dissection
  • • Powerful display and capture filtering
  • • Statistical analysis and reporting
  • • Support for 1000+ protocols
  • • Conversation and endpoint analysis
  • • Expert analysis and troubleshooting
  • • Extensible with plugins and custom dissectors

Wireshark Fundamentals

Interface Overview

Main Window Components
  • • Packet List Pane: Shows captured packets in summary format
  • • Packet Details Pane: Displays protocol tree for selected packet
  • • Packet Bytes Pane: Shows raw packet data in hex and ASCII
  • • Display Filter Bar: Apply filters to view specific packets
  • • Status Bar: Shows capture statistics and information
Key Features
  • • Coloring Rules: Visual packet classification
  • • Follow Stream: Reconstruct conversation flows
  • • Expert Info: Automated problem detection
  • • Statistics: Protocol hierarchy and I/O graphs
  • • Export Objects: Extract files from captures

Your First Packet Capture

1
Start Wireshark and Select Interface

Launch Wireshark and select your network interface from the welcome screen. Look for the interface with traffic (shown by the graph lines).

2
Start Capturing

Click the blue shark fin icon or press Ctrl+E to start capturing packets. You'll immediately see packets flowing in the packet list pane.

3
Generate Some Traffic

Open a web browser and visit a website to generate HTTP traffic. Try pinging a server to see ICMP packets.

4
Stop and Analyze

Press Ctrl+E again to stop capturing. Click on individual packets to see their details in the packet details pane.

Command Line Interface: TShark

TShark is Wireshark's command-line counterpart, perfect for automation and remote analysis.

# Basic packet capture
tshark -i eth0

# Capture to file
tshark -i eth0 -w capture.pcap

# Read from file
tshark -r capture.pcap

# Apply capture filter
tshark -i eth0 -f "tcp port 80"

# Apply display filter
tshark -r capture.pcap -Y "http.request.method == GET"

Advanced Filtering Techniques

Capture Filters vs Display Filters

Capture Filters

Applied during packet capture. Use BPF (Berkeley Packet Filter) syntax. More efficient but less flexible.

# Protocol filters
tcp
udp
icmp

# Host filters
host 192.168.1.1
src host 192.168.1.1
dst host 192.168.1.1

# Port filters
port 80
src port 443
dst port 22

# Complex filters
tcp and port 80
host 192.168.1.1 and not port 22
Display Filters

Applied after capture. More powerful and flexible with protocol-specific fields.

# Protocol filters
tcp
http
dns

# IP address filters
ip.addr == 192.168.1.1
ip.src == 10.0.0.1
ip.dst == 172.16.0.1

# TCP flags
tcp.flags.syn == 1
tcp.flags.reset == 1

# HTTP specific
http.request.method == "GET"
http.response.code == 200

Essential Display Filters

Category Filter Description
Protocol tcp All TCP traffic
udp All UDP traffic
http HTTP traffic only
dns DNS queries and responses
IP Address ip.addr == 192.168.1.1 Traffic to/from specific IP
ip.src == 10.0.0.1 Traffic from specific source
ip.dst == 172.16.0.1 Traffic to specific destination
Port tcp.port == 80 TCP traffic on port 80
tcp.srcport == 443 TCP traffic from port 443
tcp.dstport == 22 TCP traffic to port 22
TCP Flags tcp.flags.syn == 1 TCP SYN packets
tcp.flags.reset == 1 TCP RST packets
tcp.flags.fin == 1 TCP FIN packets

Zenmap: Nmap's Graphical Interface

What is Zenmap?

Zenmap is the official graphical user interface (GUI) for the Nmap Security Scanner. It provides an intuitive, visual way to use Nmap's powerful features while maintaining access to all command-line functionality.

Key Features

  • • Visual network topology mapping
  • • Pre-configured scan profiles
  • • Scan result comparison tools
  • • Interactive host and service details
  • • Real-time scan progress monitoring
  • • Comprehensive result filtering
  • • Command wizard for complex scans
  • • Export capabilities for reporting

Getting Started with Zenmap

Interface Overview

Main Components
  • • Target Field: Enter IP addresses or hostnames
  • • Profile Dropdown: Select pre-configured scan types
  • • Command Field: Shows/edit the actual Nmap command
  • • Scan Button: Execute the scan
  • • Results Tabs: View scan output in different formats
Result Views
  • • Nmap Output: Raw command-line output
  • • Ports/Hosts: Structured port and service information
  • • Topology: Visual network map
  • • Host Details: Comprehensive host information
  • • Scans: History of previous scans

Use Zenmap When:

  • • Learning Nmap functionality
  • • Need visual network topology
  • • Comparing scan results
  • • Creating reports with screenshots
  • • Exploring scan results interactively
  • • Working with non-technical stakeholders
  • • Quick ad-hoc scanning
  • • Demonstrating network discoveries

Use Nmap CLI When:

  • • Automation and scripting
  • • Remote server administration
  • • Continuous integration pipelines
  • • Maximum performance needed
  • • Complex custom commands
  • • Minimal system resources
  • • Batch processing large networks
  • • Integration with other tools

Pro Tip

Use Zenmap to learn and experiment with Nmap commands, then transition to CLI for production work. Zenmap's command field shows exactly what Nmap command is being executed, making it an excellent learning tool.
